⚙ Setup instructions

1 — Create a Strava API Application

  1. Go to strava.com/settings/api and create an app.
  2. Set Authorization Callback Domain to your Netlify site domain (e.g. your-site.netlify.app).
  3. Note your Client ID and Client Secret.

2 — Deploy to Netlify

  1. Push this project to a GitHub repo.
  2. In Netlify, click Add new site → Import from Git, select the repo.
  3. In Site settings → Environment variables, add:
    STRAVA_CLIENT_ID = your client ID
    STRAVA_CLIENT_SECRET = your client secret
  4. Trigger a redeploy so the env vars take effect.

3 — Garmin Connect

Enter your Garmin email and password directly — no API keys needed. Credentials travel over HTTPS and are never stored. If Garmin asks for 2FA, tick Have 2FA? to reveal the MFA field.

Notes

  • The Garmin function may time out on Netlify's free tier (10 s limit). Netlify Pro raises this to 26 s.
  • Garmin's SSO occasionally blocks automated logins. If you get repeated auth failures, wait 30 minutes and retry.
  • Power data in Strava streams requires a running power meter or compatible device/app (e.g. Stryd).
